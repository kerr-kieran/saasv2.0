# 单元测试规范

## 框架

- JUnit 5 + Mockito + AssertJ
- 覆盖率工具: JaCoCo

## 目标覆盖率

| 层级 | 行覆盖率 | 分支覆盖率 |
|------|---------|-----------|
| Service | >= 90% | >= 80% |
| Controller | >= 70% | >= 60% |
| Mapper | >= 60% | >= 50% |
| 工具类 | >= 95% | >= 90% |

## 命名规范

`should{ExpectedBehavior}When{Condition}`

示例: `shouldCreateOrderSuccessfullyWhenStockSufficient`

## 结构

```java
@Test
@DisplayName("中文描述")
void shouldXxxWhenYyy() {
    // Given — 准备数据和Mock
    // When — 执行被测方法
    // Then — 断言结果 + 验证Mock调用
}
```

## 原则

- 每个测试方法只测一个场景
- 不依赖执行顺序
- Mock 所有外部依赖
- 核心业务路径 100% 覆盖
