# 业务流程

## 订单全生命周期

```
消费者在抖音/淘宝下单
        │
        ▼
  ERP 自动抓单
        │
        ▼
  智能审单（风控检查、库存检查）
        │
        ├── 风控不通过 → 人工审核 → 通过/拒绝
        │
        ▼
  拆单合单（按仓库、供应商拆分/合并）
        │
        ▼
  按分仓规则推送给 WMS
        │
        ▼
  WMS 生成波次拣货
        │
        ▼
  拣货 → 复核 → 打包
        │
        ▼
  物流发货
        │
        ├──── 库存自动扣减并同步回 ERP
        │
        ▼
  财务生成结算数据
        │
        ▼
  CRM 同步订单信息
        ├── 触发会员积分/营销活动
        │
        ▼
  售后处理（退货/换货/退款）
        │
        ▼
  订单完成 → 订单生命周期结束
```

## 订单状态机

```
PENDING ──► PAID ──► SHIPPED ──► COMPLETED
   │           │         │
   └──► CANCELLED  ◄─────┘
              (已付款可取消，需退款+库存释放)
```

| 状态 | 值 | 说明 |
|------|-----|------|
| PENDING | 0 | 待支付 |
| PAID | 1 | 已支付（待发货） |
| SHIPPED | 2 | 已发货 |
| COMPLETED | 3 | 已完成（确认收货） |
| CANCELLED | 4 | 已取消 |

## 库存扣减流程

```
下单请求 → 锁定库存(Redis分布式锁) → 检查库存 → 扣减库存
                                          │
                                    ┌─────┘
                                    ▼
                             库存不足 → 订单失败
                                    │
                                    ▼
                             库存充足 → 创建订单 + 发送MQ(INVENTORY_DEDUCT)
                                         │
                                         ▼
                                   库存服务消费MQ → 真实扣减DB库存
```

## 支付回调流程

```
支付平台回调 → Gateway → Payment Service
                            │
                  验证签名  ◄─┘
                            │
                  更新支付记录 → 发送MQ(ORDER_STATUS:paid)
                            │
                  订单服务消费 → 更新订单状态 → 发送MQ(MEMBER_POINTS:order_pay)
                                                                  │
                                                        会员服务消费 → 发放积分
```
