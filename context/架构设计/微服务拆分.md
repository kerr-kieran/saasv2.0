# 微服务拆分

## 服务矩阵

| 服务名 | 职能 | 核心领域对象 | 数据库 | 依赖服务 |
|--------|------|-------------|--------|---------|
| saas-gateway | API网关：统一入口、鉴权、限流、路由 | Route, Filter | - | Nacos |
| saas-user-service | 用户注册、登录、登出、token管理 | User, UserToken | saas_user | Redis |
| saas-product-service | 商品浏览、搜索、分类、详情 | Category, SPU, SKU | saas_product | Redis |
| saas-order-service | 订单创建、支付、取消、查询 | Order, OrderItem | saas_order | user, product, inventory, payment, Redis, RocketMQ |
| saas-inventory-service | 库存查询、扣减、释放、仓库管理 | Inventory, Warehouse, StockLog | saas_inventory | RocketMQ, Redis |
| saas-member-service | 会员信息、地址、积分、等级 | Member, Address, PointsLog | saas_member | RocketMQ, Redis |
| saas-payment-service | 支付处理、回调、退款 | Payment, Refund | saas_payment | order, Redis |
| saas-admin-service | 后台管理聚合：商品/订单/会员/库存/权限管理 | (聚合层) | saas_admin | 所有业务服务 |

## 服务间通信

| 调用方 \ 被调方 | user | product | order | inventory | member | payment |
|-----------------|------|---------|-------|-----------|--------|---------|
| gateway | REST | REST | REST | REST | REST | REST |
| order | REST | REST | - | REST + MQ | - | REST |
| inventory | - | - | MQ | - | - | - |
| member | - | - | MQ | - | - | - |
| admin | REST | REST | REST | REST | REST | REST |

## 拆分原则

1. **DDD限界上下文**：按业务域拆分，每个服务对应一个限界上下文
2. **数据库每服务**：每个微服务独立数据库，禁止跨服务直连数据库
3. **同步REST + 异步MQ**：核心写链路用REST保证强一致，非核心链路用MQ异步解耦
4. **无状态设计**：所有业务服务无状态，会话信息存Redis
5. **独立部署**：每个服务独立构建、测试、部署
